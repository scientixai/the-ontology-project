projectionproduces
adam_traceability.rqADaM traceability — walk an analysis result to its NATIVE source fact (the lineage leaf). "Reproduce this number" = a prov:wasDerivedFrom+ walk to the fact that has no further source.
certified_copy_register.rqCertified-copy register — each certified copy and the source document it reproduces, with who certified it. The E6(R3) certified-copy chain as a query (an inspector's "show me the original behind this scan" is a lookup, not a hunt).

deviation_lineage.rqDeviation lineage — trace a CAPA back through the deviation it answers to the latent antecedent and the critical-to-quality factor that antecedent threatened. "Why does this CAPA exist, and what quality factor was ultimately at stake" is a query, not an audit dig.
doa_log.rqDelegation of Authority (DOA) log — PROJECTED from delegation acts. The site-side flagship: the "log" is a query over the graph, never hand-maintained.
fhir_research_subject.rqFHIR ResearchSubject view, projected from HCLS-core/CR-core — the SAME native graph yielding a FHIR-shaped view (vs the CDISC views), proving one model many edges.
planned_vs_actual.rqPlanned-vs-actual reconciliation — the join a static SoA cannot express. For each realized visit, the planned target and resolved window beside what actually happened, so out-of-window adherence is a projection, not a manual review.
rbqm_monitoring.rqRBQM monitoring allocation — for each critical-to-quality factor, its risk level and the resulting SDV intensity. The risk-targeted monitoring table, projected from the graph.
sdtm_ae.rqCDISC SDTM — AE (Adverse Events) domain, projected from the native graph. Covers AdverseEvent and its subclasses (e.g. DoseLimitingToxicity) via subclass path.
sdtm_dm.rqCDISC SDTM — DM (Demographics) domain, projected from the native operator graph. One row per enrolled subject. Standards are views, not the stored model.
sdtm_ex.rqCDISC SDTM — EX (Exposure) domain, projected from dosing acts (cr:Administration).
site_activation_tracker.rqStart-up tracker — the activation timeline per site (IRB -> CTA -> SIV -> activated), projected from the milestone graph.
soa_table.rqSchedule of Activities — the classic table is just a VIEW of the timeline. Projects each participant's planned visits back into the familiar visit x date grid. The timeline is the truth; this table is one read of it (no footnotes — the timing logic is in the windows).
specimen_lineage.rqBlood-draw traceback — an assay result -> sample -> (aliquot/derivation)* -> primary specimen -> the collection event. "Trace this result to the draw" is a query.
sssom_export.rqSSSOM TSV export — projected from the owned crosswalk mappings. The SSSOM table is a VIEW of the curated mappings, not a separate source of truth.
tmf_binding_map.rqTMF binding map — for each TMF artifact type, the domain class its facts are bound to. Once a system has classified a document to its artifact type, this tells it which ontology entities to expect inside. The comprehension layer the eTMF world omits, as a query.
