Visit execution

Two chains meet at Visit: the design chain (Protocol → SOA → VisitDefinition → ActivityTemplate) carries what is supposed to happen; the execution chain (StudySite → VisitOccurrence → ActivityOccurrence → ClinicalObservation) carries what actually happened. TOP separates them; most trial systems conflate them.

Design boundary

USDM v4 Encounter is template-only (verified against cdisc-org/usdm/main/src/usdm_model/encounter.py): no per-occurrence runtime data. VisitDefinition is USDM-ingest-derivable; VisitOccurrence is execution-side (created by EDC / scheduling system when a Participant attends a real visit).

VisitOccurrence anchors at both Participant AND StudySite (not via traversal). Query convenience: "all visits at MSKCC across all participants" requires no join.

UNSCHEDULED visits (urgent care, AE workup, sponsor-requested follow-up) have no VisitDefinition; definedBy is absent.

ActivityOccurrence — universal substrate pattern

Any assessment — vitals, MRI, ePRO, biopsy, IP administration — uses the same entity shape. Content specialisation is via biomedicalConceptCode and taskValueType. This avoids a proliferation of activity-type subclasses while preserving full SDTM-domain fidelity at projection time.

AttributeNotes
activityStatusPLANNED / PERFORMED / SKIPPED / FAILED. NGSI-LD temporal property.
biomedicalConceptCodeThe CDISC COSMoS BC identifier driving SDTM domain assignment (VS, LB, EG, EX, PE …).
taskValueTypeNUMERIC / TEXT / DATE / CODED / BOOLEAN. Determines the observation structure.
plannedStartDate / actualStartDateDesign vs execution timestamps; bitemporal via NGSI-LD observedAt.
skipReasonOperator-grounded reason when activityStatus=SKIPPED.
instantiatesTemplate→ ActivityTemplate · 0..1. Absent for unscheduled activities.
performedBy→ hcls:Person · 0..1. Must be covered by a valid Delegation (enforced by cr:CapabilityDelegationShape).
hasObservation→ ClinicalObservation · 0..N. Structured results.

SHACL invariants (Inv #52–#58)

#SeverityConstraint
52ViolationVisitOccurrence requires Participant. forParticipant must be present (1..1).
53ViolationVisitOccurrence requires StudySite. atStudySite must be present (1..1).
54ViolationStudySite consistency. atStudySite must match forParticipant.forStudySite.
55WarningCOMPLETED visit should have actualEndDate.
56WarningUNSCHEDULED visit should have reason. unscheduledReason should be set when visitType=UNSCHEDULED. Maps to SDTM SV SVUPDES.
57WarningOut-of-window visit should link to Deviation. windowStatus=OUT_OF_WINDOW should have an associatedDeviation per ICH E6(R3) §4.5.
58ViolationClinicalObservation must use pseudonymous subject. observationForSubject must be a cr:StudySubject, never an hcls:Person (PII containment, ADR-0019).

Query patterns

GET /ngsi-ld/v1/entities?type=VisitOccurrence
  &q=forParticipant=="urn:ngsi-ld:Participant:onco-423-mskcc-p001"
    ;visitStatus=="COMPLETED"
# Coordinator: all completed visits for Mary

GET /ngsi-ld/v1/entities?type=VisitOccurrence
  &q=atStudySite=="urn:ngsi-ld:StudySite:mskcc-onco423"
    ;windowStatus=="OUT_OF_WINDOW"
# Monitor: out-of-window visits at MSKCC requiring deviation docs

GET /ngsi-ld/v1/entities/urn:ngsi-ld:VisitOccurrence:v-001?join=inline&joinLevel=3
# Coordinator screen: full visit with inline ActivityOccurrences + ClinicalObservations

Open issues

  • Observation lift: ClinicalObservation is modeled but the full SDTM-domain projection rules (VS vs LB vs EG vs PE based on biomedicalConceptCode) need a projection map per domain. Defer to when COSMoS BC catalog integration lands.
  • Appointment as pre-visit artifact: FHIR R5 distinguishes Appointment (pre-occurrence scheduling) from Encounter (occurred). TOP currently collapses both into VisitOccurrence via status=SCHEDULED. If sponsors need separate scheduling artifact, lift cr:VisitAppointment as a sub-object.
  • Digital-twin surface: when ClinicalObservation carries time-varying measurements (vitals over time, lab trajectories), it becomes the primary twin-queryability surface. See participant.html §Twin-for-enrollment context.
